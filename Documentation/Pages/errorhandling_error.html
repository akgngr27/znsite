Error(Hata) Kütüphanesi

Sınıflarda istenilen hata verilerinin tutulması ve kullanılması için oluşturulmuştur. Sınıfların hata verilerinin tutulması için ZN'in bir çok kütüphanesinde kullanıldığı için bu kütüphanelerin kullanımında oluşan hataları öğrenmek içinde kullanabilirsiniz.

Hata Verisi Oluşturmak

Errors::set(string $hataMesaji )

Hata verisi oluşturmak için kullanılır. Kullanılan sınıfa ait yöntemin ismini otomatik olarak almaktadır. Bunun için sizin sınıf veya fonksiyon ismi bildirmenize gerek yok. Sadece hata mesajını yazmanız yeterlidir.

ParametreAnlamları
1. Parametre = Hata MesajıHangi hatanın tutlacağı.

class TestClass
{
    public static function yontem($p1)
    {
        if( ! is_string($p1) )
        {
            Errors::set('Geçersiz parametre kullanımı!'); 
        }
    }
}

Hata Verisini Kullanmak

Errors::get( string $sinifAdi, string $yontemAdi, [ string $tur = 'string' ] )

Hata verisi oluşturmak için kullanılır.

ParametreAnlamları
1. Parametre = Sınıf AdıHatanın hangi sınıf içinde oluşturulacağı belirtilir.
2. Parametre = Yöntem AdıHatanın hangi yöntemi kapsadığı belirtilir.
[3. Parametre = Mesaj Türü = string]Hata mesajının içeriği. Alacağı Değerler: string, array ve table
3. Parametrenin Alabileceği Değerler
stringMetinsel çıktı döndürür. Herhangi bir hata oluşmamışsa false değeri döner.
arrayDizisel çıktı döndürür. Herhangi bir hata oluşmamışsa false değeri döner.
tableTablolu çıktı döndürür. Herhangi bir hata oluşmamışsa hata olmadığına dair mesaj değeri döner.

TestClass::yontem();
echo Errors::get('TestClass', 'yontem'); // TestClass::yontem : Geçersiz parametre kullanımı!
output(Errors::get('TestClass', 'yontem', 'array')); // Geçersiz parametre kullanımı!

Önceden tanımlanmış kütüphanelerin kullanımında oluşan kullanım kaynaklı hataları öğrenmek içinde kullanılabilir.

DB::get(array())->result;
echo Errors::get('DB'); // DB::get : Table parametresi string ifade içermelidir!

Hata Verisini Metinsel Türde Almak

Errors::getString( string $sinifAdi, string $yontemAdi)

Hata verisini metinsel türde almak için kullanılır. Hata oluşmuşsa hatanın değeri oluşmamışsa false değeri döner.

ParametreAnlamları
1. Parametre = Sınıf AdıHatanın hangi sınıf içinde oluşturulacağı belirtilir.
2. Parametre = Yöntem AdıHatanın hangi yöntemi kapsadığı belirtilir.

Encode::super();
echo Errors::getString('Encode', 'super'); // Encode::super : Geçersiz parametre kullanımı!

Hata Verisini Dizisel Türde Almak

Errors::getArray( string $sinifAdi, string $yontemAdi)

Hata verisini dizisel türde almak için kullanılır. Hata oluşmuşsa dizi oluşmamışsa false değeri döner.

ParametreAnlamları
1. Parametre = Sınıf AdıHatanın hangi sınıf içinde oluşturulacağı belirtilir.
2. Parametre = Yöntem AdıHatanın hangi yöntemi kapsadığı belirtilir.

Encode::super();
output(Errors::getArray('Encode', 'super')); // Array

Hata Verisini Tablo ile Almak

Errors::getTable( string $sinifAdi, string $yontemAdi)

Hata verisini tablo şeklinde almak için kullanılır. Hata oluşsada oluşmasada bir tablo çıktısı döner.

ParametreAnlamları
1. Parametre = Sınıf AdıHatanın hangi sınıf içinde oluşturulacağı belirtilir.
2. Parametre = Yöntem AdıHatanın hangi yöntemi kapsadığı belirtilir.

Encode::super();
echo Errors::getTable('Encode', 'super'); // Table

Tablolu Hata Çıktısı Bastırmak

Errors::message( string $dilDosyasiAdi, string $anahtar, mixed $extra)

lang() ya da getErrorMessage() yönteminin kullanımı ile aynıdır. Dil dosyalarında yer alan mesajları özel bir tablo ile çıktılamak için kullanılır.

echo Errors::message('Error', 'arrayParameter', '1.(test)'); // "1.(test) parametresi array veri türü içermelidir" mesajını tablolu bir göderim ile ekrana basar.

Hangi mesajları bastırabileceğinizi görmek için System/Languages/ dizini içinde yer alan dil dosyalarına bakabilirsiniz. Kendi tanımlacağınız dil dosyaları için Application/Languages/ dizinide kullanılabilir.
