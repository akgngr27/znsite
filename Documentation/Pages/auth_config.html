User(Kullanıcı) Ayarları

Kullanıcıların kayıt, giriş aktivasyon gibi işlemlerinin gerçekleşmesi için veritabanı bağlantısı kuracak bir kaç sütun ve tablo bilgilerini içeren dosyadır.

Config/User.php dosyasında yer alan ayarların nasıl yapılması gerektiğine aşağıdaki tabloda yer verilmiştir.

Ayar AdıVarsayılanİşleviÖrnek Kullanım
encode string empty Şifre verisinin şifreleme türünü belirtmek için kullanılır. Bu ayar ayarlı iken şifreyi tekrar kodlamaya gerek yoktur. Alabileceği değerler: super, golden, md5, sha1 ... ve diğer hash algoritmaları. Örnek: 'md5'
matching->table string empty User kütüphanesinın bağlantı kurulacağı kullanıcı tablosu adı. Örnek: 'kullanicilarTablosu'
matching->columns->username string empty Yukarıda belirtilen kullanıcı tablosunda kullanıcı adını tutan sütun adı. Örnek: 'kullaniciAdi'
matching->columns->password string empty Yukarıda belirtilen kullanıcı tablosunda kullanıcı şifresini tutan sütun adı. Örnek: 'sifre'
matching->columns->email string empty Kullanıcı e-postasını tutan sütun adı. Bu sütunun kullanımı zorunlu olmamakla birlikte şayet kullanıcı adı sütünu e-posta bilgisi içermiyorsa bu sütun kullanılmalıdır. Yani tablonuzda en az bir e-posta sütünu olmalıdır. Bu sütun kullanıcı adını tutan sütun değilse bunun yerine bu veriyi tutacak e-posta sütunu hangisi ise o sütun bilgisi girilir. Örnek: 'epostalar'
matching->columns->active string empty Kullanımı zorunlu değildir. Kullanıcıların aktif olma durumunu kontrol eder. Kullanıcılardan hangisinin aktif olup hangisinin aktif olmadığını öğrenmek için kullanılır. Bu sütunu kullanmak için kullanıcı tablosunda bu bölüme uygu sütun oluşturup bu bölüme yazılmadılır. Kullanılmayacaksa boş bırakılmalıdır. Kullanılacaksa bu sütunun alacağı veri türü 0 ve 1 değeri içerecek türde olmalıdır. Örnek: 'aktifDurum'
matching->columns->banned string empty Kullanıcıların banlı olma durumunu kontrol eder. Bu sütun aktifse banlanmış kullanıcılar siteye giriş yapamaz. Kullanıcılardan banlananların siteye girememesini sağlar. Bu sütun kullanılmayacaksa boş bırakılmalıdır. Kullanılacaksa bu sütunun alacağı veri türü 0 ve 1 değeri içerecek türde olmalıdır. Örnek: 'banDurum'
matching->columns->activation string empty Üyelik işlemlerinde aktive zorunluluğu olmasını isterseniz kullanıcı tablosunda bu alana uygun bir sütun oluşturulmalı ve sütun adı bu bölüme eklenmelidir. Aktivasyon işlemi istenmiyorsa bu sütun boş bırakılmalıdır. Sütun Veri Türü: Char / Int. 0 veya 1 değeri tutmalıdır. 1 Aktivasyon Yapılmış, 0 Aktivasyon Yapılmamış. Örnek: 'aktivasyonDurum'
joining->column string empty Kullanıcılarla iglili birden fazla tablo sayısı varsa yukarıda belirtilen tabloya ait birleştirme için kullanılan sütun ismi belirtilir. Bu sütun genellikle ID değerini tutan sütundur. Örnek: 'ID'
joining->tables array empty Kullanıcılarla ilgili farklı tablolarla birleştirme işlemi yapılacağı zaman kullanılır. Bu ayara dizi türünde veri girilir. Dizinin anahtarı olarak birleştirlecek diğer tabloların adı, değer olarakta o tablolara ait birleştirmede kullanılan sütün adı girilir. Örnek: array('tablo1' => 'birlestirilecekSutun1', 'tablo2' => 'birlestirilecekSutun2')
emailSenderInfo array empty Aktivasyon işlemleri veya şifremini unuttum işlemleri esnasından gönderilecek e-posta'ya ait gönderen ismi ve e-posta bilgilerini belirtmek içindir. Genellikle site adı ve siteye ait e-posta adresi tercih edilir. Örnek: array('name' => 'Site Adı', 'mail' => 'info@site.com')
