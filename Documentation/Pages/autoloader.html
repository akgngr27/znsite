Autoloader Kullanımı

Autoloader zeroneed framework'te tanımlanan sınıf yapılarının dahil edilmeksiniz doğrudan kullanımını sağlar. Böylece sizler hem kişisel hemde farklı kaynaklardan taşıyacağınız kütüphaneleri rahatlıkla entegre edebilirsiniz. Autoloader, Trait, Interface, Abstract Class ve Namespace'lerin de kullanımına izin verir.

Autoloader'i Çalıştırmak

Autoloader::run( string $sinifAdi )

Bu yöntem sistem tarafından otomatik olarak çağırılmaktadır. Otomatik yükleyici çalıştırılarak Config/Autoloader.php dosyasında yer alan classMap ayarına belirtilen dizinlerde tarama yapılır. Yeni bir sınıf bulunmuş ise Config/ClassMap.php dosyasına ilave eder.

Class Map(Sınıf Haritası) Oluşturmak

Autoloader::createClassMap()

Bu yöntem sistem tarafından otomatik olarak çağırılmaktadır. Config/ClassMap.php dosyasına Sınıf isimleri ve İsim alanları bilgilerinin oluşturulmasını sağlar.

Sınıf Adı Belirtilen Sınıf Hakkında Bilgi Almak

Autoloader::getClassFileInfo( string $sinifAdi )

Sınıf adı belirtilen yapıya ait sınıf, isim alanı ve dosya yolu bilgilerine ulaşılır.

query()
ParametreKullanımı
string Sınıf AdıDönen bilgiler. path, class, namespace

output( Autoloader::getClassFileInfo('DB') );

/*
path => string 'System/StaticAccess/Database/DB/__USE_STATIC_ACCESS__DB.php' ( length = 61 ),
class => string 'DB' ( length = 4 ),
namespace => string 'DB' ( length = 4 ),
*/

Sınıf Yolu Belirtilen Sınıf Hakkında Bilgi Almak

Autoloader::tokenClassFileInfo( string $sinifYolu )

Sınıf yolu belirtilen yapıya ait sınıf ve isim alanı bilgilerine ulaşılır.

query()
ParametreKullanımı
string Sınıf AdıDönen bilgiler. class, namespace

output( Autoloader::tokenClassFileInfo('System/StaticAccess/Database/DB/__USE_STATIC_ACCESS__DB.php') );

// class => string 'DB' ( length = 4 )

Autoloader'i Yeniden Başlatmak

Autoloader::restart()

Bu işlem genellikle istisnai kullanımlardan kaynaklı sınıfların otomatik oluşturulamamasında kullanılır. Bu istisnai durumun oluşması ise şöyledir;

İstisna-1 Daha önce oluşturulmuş bir sınıfın dosya ismi değiştirilmeden dosya içindeki class adının değiştirilmesi. Hali hazırda daha önceden oluşturulmuş böyle bir dosyanın varlığı söz konusu olduğu için yeniden oluşturma işlemi yapılamayacaktır.

İstisna-2 Libraries/Test.php dosyasının sınıf adı Test olarak oluşturulup kullanıldıktan sonra bu sınıf adının başına __USE_STATIC_ACCESS__ ön ekinin getirlmeye çalışılması durumunda da oluşturma başarısızlıkla sonuçlanacaktır. Bunun nedeni yine İstisna-1 de anlatılan durumla aynıdır.

Sonuç: Bu gibi nedenlerle sınıf yeniden oluşturulamıyorsa Sınıf kullanımından önce Autoloader::restart() yöntemi çalıştırılır.

Autoloader::restart();

Test::run();

Autoloader ile OOP KUllanımı Hakkında

ZN framework'ün sınıflarında namespace kullanımı yoktur. Günümüz uygulamalarında oldukça gerekli görülen ve bizimde öyle düşündüğümüz isim alanlarının ZN'de tercih edilmemesinin sebebi mevcut Autoloader kütüphanesi ile kolaylık sınıfların harita yol bilgilerinin oluşturulmasıdır. Ancak bizlerin isim alanı kullanmamamız sizin de kullanmayacağınız anlamına gelmez bu nedenle ZN Framework kullanıcıların kütüphane geliştirilmelerinde mevcut yeni teknolojilerin kullanımına imkan verir. Trait, Interface, Abstract Class, Namespace.

Şayet kütüphanelerinizde namespace kullanırsanız Config/ClassMap.php dosyasında sınıf isimleri ile birlikte isim alanlarıda eklenir. İsim alanı ile sınıf oluşturmuşsanız ve o sınıf ismi ile aynı başka bir sınıf ismi mevcut değilse. Doğrudan isim alanı olmasına rağmen $this->sınıf->yontem() şeklinde kullanabilirsiniz. Böylece isim alanlarını harfi harfine yazmak zorunda değilsiniz. Ancak bu kural statik kullanımda mümkün değildir.
