Multi Byte(MB) Kütüphanesi

Bu kütüphane PHP'nin mb_ yöntemlerini kullanarak sınıf biçimine dönüştürülerek oluşturulmuştur. İşinize yaracağınız düşündüğümüz yöntemleri kullandık. İsimlendirmeleri akılda kalacak şekilde düzenlemeye özen gösterilmiştir.

0 - 255 arasıdan değere sahip olan karakterler dışındaki karakterlerin kullanımı her zaman sorunlar oluşturmaktadır. Mesela substr() yönteminde normal karakter dışındaki karakterlerin kesitlenmesinde karakter hataları oluşmaktadır. Bu sorunu aşmak için mb_substr() yöntemi kullanılır. İşte bizde mb_ yöntemlerinden sık kullanılması düşünülen yöntemleri bir araya getirdik.

Kullanım Farklılığı Olan Yöntemler

Search() yöntemi string yöntemlerin bir kaçının bir araya getirilerek oluşturulmuş halidir. Hangi yöntemleri kullandığı yukarıda belirtilmiştir. Şimdi bu yöntemin kullanımına dair bir örnek verelim. 

/*
@param string -> Arayamaya tabi tutulacak metinsel ifade
@param string -> Aranılan veri
@param string -> Yöntem: str/string, pos/position -> str seçilirse strstr(), pos seçilirse strpos() yöntemi kullanılmış olur. Varsayılan:str
@param bool   -> Duyarlılık: true -> Büyük-küçük harf duyarlılığını kaldırmak isterseniz false olarak ayarlamalısınız.
*/

MB::search('ZN Framework 2', 'Framework'); // Çıktı: Framework 2

// 4. parametrenin false olarak ayarlanması ile harf duyarlılığını kaldırmış olduk.
MB::search('ZN Framework 2', 'framework', 'str', false); // Çıktı: Framework 2

// Şimdide strpos() yöntemi ile aram işlemi yapalım. 3. parametre pos/position olarak ayarlanıyor.
MB::search('ZN Framework 2', 'framework', 'pos', false); // Çıktı: 3

mb_convert_case() yönteminin kullanımında da küçük fark var. MB case sabitlerinde yer alan MB_CASE_ ön eki kullanılmaz. 

// MB_CASE_ sabiti kullanılıyor.
mb_convert_case('çevir', MB_CASE_UPPER);

// MB_CASE_ sabiti kullanılmıyor.
MB::casing('çevir', 'upper');

Diğer yöntemlerin kullanımları genel olarak aynıdır.

Data detaylı kullanım için multi byte php.net kaynaklarını inceleyiniz.
