DataGrid(Veri Tablosu) Kütüphanesi

Veritabanı tablolarına ait verileri listelemek, veriler üzerinde silme, ekleme ve düzenleme işlemleri gerçekleştirmek için oluşturulmuştur. Bu işlemlerden güncelleme ve ekleme işlemleri Tekil yani ilişkisiz tablolar için geçerlidir. Görüntüleme ve silme işlemi için ilişkisel yöntemler kullanılabilir.

DataGrid Kütüphanesi Yöntemleri

Veri tablosunu oluşturmak için bazı yöntemlerden yararlanılır.

DataGrid Kütüphanesi YöntemleriAnlamları
table(string $tabloAdi) Hangi tablonun verileri üzerinden çalışılacaksa o tablonun isim bilgisidir.
processColumn(string [$sutun = 'id'], boolean [$duzenlenebilir = false]) Ekleme, silme ve güncelleme işlemlerinin hangi sütuna göre yapılacağı belirtilir. Bu değer ayarlanmaz ise varsayılan olarak id değeri kabul edilir. 2. parametre ise bu sütunun da düzenlenebilir düzenlenemeceğin belirlenmesi için kullanılır. Varsayılan olarak düzenlemeye kapalıdır. Bu parametre true olarak ayarlanırsa düzenlenebilir hale gelir.
limit(numeric [$limit = 20]) Tek bir sayfada listelenmesi istenen kayıt miktarını ayarlamak için kullanılır. Bu değer ayarlanmaz ise varsayılan olarak 20 değeri kabul edilir.
columns([array $sutunlar]) Grid'de hangi sütunların görüntüleneceğini belirtmek için kullanılır. Bu yöntem kullanılmaz ise tabloya ait tüm sütunlar Grid'de görüntülenir. Bu yöntemle aynı zamanda gösterilecek sütunlara ilişkin takma ad, Form nesnelerine ait input türleride ayarlanır.
Ayarlanabilir SeçeneklerAnlamları
aliasTakma isim
titleSütun İsimleri
inputForm input türü
where(array($kosul, $deger, $baglac) ... ) Where koşulunu oluşturmak için kullanılır. Sıralı dizi parametreleri alır. Birden fazla where yapısı kullanmak için parametre sayısı artırılabilir.
orderBy([string $sutun], [string $tur = DESC]) Ön tanımlı sıralama oluşturmak için kullanılır. Aksi halde zaten grid'de sütun isimlerine tıklandığında a-z, z-a sıralama yapmaktadır.
groupBy([string $sutun]) Gruplama yapmak için kullanılır.
joins(array($tabloAdiveSutun, string $esasTabloSutun ... )) Esas tablo ile birleştirmek yapmak için kullanılır. 1 parametre Birleştirilecek diğer tabloya adit tablo adı ve sütun bilgisi içerir. Örnek: tablo.id = id. Birden fazla tabloyu ilişkilendirmek için diziye birleştirme değerleri eklenerek kullanılabilir. Ancak birleştirme işlemi esas tabloya göre yapılmaktadır. Join yöntemi columns() yönteminin kullanılmasını gerektirir. Aynı isimli sütun isimlerinin çakışmaması için alias yani takma isimlerden yararlanılır.
create() DataGrid kodu oluşturma yöntemidir. En son bu yöntem kullanılarak DataGrid'in oluşturulması sağlanır.

Yukarıdaki yöntemlerin kullanımına yönelik örnekler aşağıda verilmiştir.

// Verileri listelenecek tablo seçiliyor.
echo DataGrid::table('tablo')
       // İşlemin yapılacağı sütun seçiliyor.
       // Kullanımı zorunlu değildir. Varsayılan: id
       ->processColumn('dataId') 
       // Tek bir sayfada görüntülenecek kayıt sayısı belirtiliyor.
       // Kullanımı zorunlu değildir. Varsayılan: 20
       ->limit(10) 
       // Tabloda görüntülenmesi istenilen sütun isimleri belirtiliyor.
       // Kullanımı zorunlu değildir.
       ->columns(array
       (
           'dataId'  => array('alias' => 'adminDataId', 'title' => 'Admin ID') ,
           'name'    => array('alias' => 'adminName', 'title' => 'Admin Name'),
           'address' => array('alias' => 'adminAddress', 'title' => 'Admin Address', 'input' => 'textarea')
       )) 
       ->create();

İlişkisel tablolar kullanılmak isternirse bazı kuralların dikkate alınarak ilişki kurulması gerekmektedir. Join işlemleri soldan birleştirme olacak şekilde yapılmaktadır. Bu nedenle birleştirme yapılırken bu kriterlerin dikkate alınması gerekmektedir. Şimdi aşağıdaki örnek üzerinden nasıl kullanılacağı noktasında bilgi verelim.

// FROM deyiminin karşılığı olan esas tablo ismi.
echo DataGrid::table('uyeler')
  		// Tek bir sayfada görüntülenecek kayıt sayısı belirtiliyor.
       ->limit(10) 
       // Dizi içindeki anahtar birleştirilecek tablo ve o tabloya ait sütun bilgisi içerir.
       // Dizi içindeki değer ise birleştirlecek esas tablonun sütun bilgisini içerir.
       // Aşağıdaki kullanımın karşılığı: uyeler_adres.uye_id = uyeler.id.
       ->joins(array('uyeler_adres.uye_id' => 'id'))
       // Gridde hangi sütunların yer alacağı belirtiliyor. 
       ->columns(array
       (
           'uyeler.id'  => array('alias' => 'uyelerId', 'title' => 'ID') ,
           'uyeler.isim'    => array('alias' => 'uyelerIsim', 'title' => 'İSİM'),
           'uyeler_adress.adres' => array('alias' => 'uyelerAdres', 'title' => 'ADRES', 'input' => 'textarea')
       )) 
       // Esas tabloya ait işlemlerin yapılacağı sütun bilgisi.
       // Id sütunlarının karışmaması için takma isim kullanılır.
       ->processColumn('uyelerId') 
       ->create();

NOT
İlişkili tablolarda listeleme ve silme işlemleri yapılabilir. Ekleme ve güncelleme işlemleri yapılamamaktadır.
