Ön Bilgiler

GD Kütüphanesinde renklerin kullanımı için imageallocate() yönteminin kullanımına gerek yoktur.

Renklerin kullanımı için R|G|B|A formunda yani 0-255|0-255|0-255|0-127 biçiminde belirtilir. Sondaki A harfi alfa değerini ifade eder.

Config/Colors.php ayar dosyasında yer alan renk isimleride kullanılabilir. white, purple, pink ...

Genellikle çizimlerde kullanılan x, y, width, height, color gibi parametreler $settings dizi parametresi altında belirtilerek kullanılır. Böylece parametre sayısın azaltılarak parametre sırasının karıştırılmasının önüne geçilmesi sağlamış oldu.

Tuval Oluşturmak

Tuval oluşturmak için parametrelerin dizilimi.

GD::canvas( numeric $genislik, numeric $yukseklik, string $color = 'transparent', bool $gercek = false )

Bir resim kaynağını kullanmak için parametrelerin dizilimi.

GD::canvas( string $kaynak )

Tuval oluşturmak için kullanılır.

ParametreAnlamları
1 - numeric GenişlikTuvalin genişlik değeri.
2 - numeric YükseklikTuvalin yükseklik değeri.
[ 3 - string Renk = transparent ]Tuvalin arkaplan rengi. Renkler 0-255|0-255|0-255 formunda ya da Config/Colors.php dosyasında belirtilen renk isimleri kullanılmalıdır. Alfa değeride belirtilmek istenirse 4. blok için 0-127 arasında değer belirtilebilir. Yani R|G|B|A formunda ayarlanır.
[ 4 - string Gerçek Renk = false ]Gerçek renklerin kullanılıp kullanılmayacağı.
ReturnObject(Nesne)

ParametreAnlamları
1 - string KaynakResim dosyasının yolu.
ReturnObject(Nesne)


/*
p1: tuvalin genişliği 200px olarak ayarlandı.
p2: tuvalin yüksekliği 400px olarak ayarlandı.
p3: tuvalin arkaplan rengi beyaz olarak ayarlandı. 255|255|255 olarakta ayarlanabilirdi.
*/

GD::canvas(200, 400, 'white')
  // generate() yöntemi zincirleme yöntemleri tamamlamak için kullanılan nihai yöntemdir.
  ->generate('png');

Harmanlama Kipini Etkinleştirmek

GD::alphaBlending( bool $kip = false)

GD::alphaBlending() gerçek renkli resimlerde iki farklı çizim kipi kullanımına izin verir. Harmanlama kipinde, tüm çizim işlevlerine rengin alfa kanalı bileşeni sağlanır. Sonuç olarak GD özdevinimli olarak, rengin çizilmesi sırasında mevcut rengin harmanlanmasını sağlar ve sonucu resimde saklar. Elde edilen piksel mattır. Harmanlamama kipinde alfa kanalı bilgisi piksel çizim rengine aynen kopyalanır. Harmanlama kipi paletli resimler için kullanılamaz.

ParametreAnlamları
[ 1 - bool Kip = false ]Harmanlama kipinin etkin olup olmayacağı; false öntanımlıdır.

GD::canvas(200, 400, 'transparent')
  ->alphaBlending(true)
  ->text('Framework', array('x' => 200, 'y' => 100, 'color' => 'blue'))
  ->generate('png');

Alfa Kanal Bilgisini Kaydetmek

GD::saveAlpha( bool $kip = true)

PNG resimleri kaydederken (tek renkli şeffaflığın tersine) alfa kanalı bilgisinin kaydedilip kaydedilmeyeceğini belirtir.

GD::saveAlpha işlevi PNG resimleri kaydederken (tek renkli şeffaflığın tersine) alfa kanalı bilgisinin tamamının kaydedilmesi için seçenek belirtir. Bunu kullanmak için alfa harmanlamayı devreden çıkarmalısınız (GD::alphaBlending(false)). Alfa kanalını her tarayıcı desteklemez. Tarayıcınız sorun çıkarırsa, betiğinizi Mozilla'nın son sürümü gibi alfa kanalına uyumlu bir tarayıcıyla yüklemeyi deneyin.

ParametreAnlamları
[ 1 - bool Kip = true ]Alfa kanalının kaydedilip kaydedilmeyeceği belirtilir; true öntanımlıdır.

GD::canvas(200, 400, 'transparent')
  ->alphaBlending(false)
  ->saveAlpha(true)
  ->text('Framework', array('x' => 200, 'y' => 100, 'color' => 'blue'))
  ->generate('png');

Resmin Kenarlarına Yumuşatma Uygulamak

GD::smooth( bool $kip = true)

Hızlı çizilen kenar yumuşatma yöntemlerini çizgiler ve içi boş çokgenler için etkin kılar. Alfa bileşenlerini desteklemez. Doğrudan harmanlama yapar. Sadece gerçek renkli resimlerle çalışır. Çizgi kalınlığı ve tarzı desteklenmez. Kenar yumuşatmalı çizim ilkellerinin şeffaf artalanda kullanılmasının sonuçları belirsizdir. Harmanlama yöntemi çizgi rengiyle artalan rengini harmanlar. Alfa bileşen desteğinin yokluğu nedeniyle alfa kenar yumuşatması yapılamaz.

ParametreAnlamları
[ 1 - bool Kip = true ]Kenar yumuşatma uygulanıp uygulanmayacağı; true öntanımlıdır.

GD::canvas(200, 400, 'transparent')
  ->smooth()
  ->rectangle(array('color' => 'blue'))
  ->generate('png');

Resme Filtre Uygulamak

GD::filter( mixed $filter, $arg1, $arg2, $arg3, $arg4)

Bir resme bir süzgeç uygular.

ParametreAnlamları
1 - mixed FiltreUygulanacak filtre.
Filtrelerİşlev
negateResimdeki tüm renkleri zıt renge dönüştürür.
grayscaleResmi gri tonlamalı hale getirir.
brightnessResmin parlaklığın değiştirir. Parlaklık seviyesini belirtmek için $arg1 değiştirgesini kullanın.
contrastResmin karşıtlık seviyesini değiştirir. Karşıtlık seviyesini belirtmek için $arg1 değiştirgesini kullanın.
colorizeRengi sizin belirtmeniz dışında grayscale gibidir. Kırmızı, yeşil ve mavi renk seviyeleri belirtmek için sırayla $arg1, $arg2 ve $arg3 değiştirgelerini, alfa kanalı için de $arg4 değiştirgesini kullanın. Ana renklerin seviyesi 0 ile 255 arasında belirtilebilir.
edgedetectResmin kenarlarını vurgulamak için kenar algılaması kullanılır.
embossResme kabartma uygular.
gaussian_blurResmi Gauss yöntemiyle bulanıklaştırır.
selective_blurResmi bulanıklaştırır.
mean_removalKaralama taslak etkisi oluşturmak için silme işlemi uygular.
smoothResmin sert hatlarını yumuşatır. Yumuşatma seviyesini belirtmek için $arg1 değiştirgesini kullanın.
pixelateResme mozaik etkisi verir. Mozaik boyutu için $arg1 mozaik etki kipini belirtmek için $arg2 değiştirgesini kullanın.
[ 2 - numeric $arg1 ]Etkilenen Filtreler: brightness, contrast, colorize, smooth, pixelate
[ 3 - numeric $arg2 ]Etkilenen Filtreler: colorize, pixelate
[ 4 - numeric $arg3 ]Etkilenen Filtreler: colorize
[ 5 - numeric $arg4 ]Etkilenen Filtreler: colorize 

GD::canvas('resim.png')
  ->filter('colorize', 0, 255, 0)
  ->filter('grayscale')
  ->filter('emboss')
  ->save('yeniResim.png')
  ->generate();
  
echo GD::result(); // Kaydedilen resmin sonuçu ekrana basılıyor.

Resmi Döndürmek

GD::rotate( float $aci, string $boslukRengi = '0|0|0', numeric $seffafligiYoksay = 0 )

Bir resmi belirtilen açıya göre döndürür. Dönüş merkezi resmin merkezi olup, döndürülen resim özgün resimden farklı boyutlara sahip olabilir..

ParametreAnlamları
